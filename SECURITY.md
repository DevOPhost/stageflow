# Security Policy

## Supported Versions

This repository is maintained as a public portfolio version of the project. Security updates and corrections are applied to the latest version available in the `main` branch.

| Version        | Supported |
| -------------- | --------- |
| Latest / main  | ✅         |
| Older versions | ❌         |

## Reporting a Vulnerability

If you identify a security issue, exposed credential, broken access flow, dependency vulnerability, or any other risk related to this repository, please report it responsibly.

You can contact the maintainer through GitHub by opening a private security advisory, when available, or by reaching out through the contact information listed in the repository profile.

Please avoid opening public issues that expose sensitive details, credentials, tokens, internal paths, or exploitable vulnerabilities.

## Public Portfolio Scope

This repository is a public portfolio version and does not include production credentials, private database access, confidential business data, or real user records.

Any sample data included in the project is used only to represent the interface and user experience of the system.

## Security Practices

This project follows basic security practices for public repositories, including:

* no `.env` files committed to the repository;
* no private API keys or service credentials;
* no production database credentials;
* no sensitive user data;
* use of `.gitignore` to avoid exposing local files;
* periodic review of dependencies and project files;
* separation between public portfolio code and private production code.

## Responsible Disclosure

Please provide enough detail to reproduce and understand the issue, including:

* affected file or page;
* steps to reproduce;
* expected behavior;
* actual behavior;
* possible impact;
* suggested correction, if applicable.

Security reports will be reviewed as soon as possible.
